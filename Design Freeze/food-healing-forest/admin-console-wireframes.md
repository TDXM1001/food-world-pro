# 食愈森林后台 MVP 低保真原型说明

## 1. 文档目标

- 固化 `MVP` 后台最小管理端的页面结构与操作流。
- 让运营、后端、测试对“后台到底最少要做什么”形成统一口径。
- 本轮只做能支撑 `日签供给 + 规则维护 + 提醒策略 + 看板 + 敏感处置` 的最小系统。

## 2. 后台导航结构

```text
后台首页
  |- 日签生成与审核
  |- 标签与提醒规则
  |- 记录与转化看板
  |- 敏感记录处置台
```

## 3. 页面 B1：后台首页总览

### 3.1 页面目标

- 让运营快速看到“供给是否正常、记录闭环是否在跑”

### 3.2 线框结构

```text
+--------------------------------------------------+
| 顶部导航 / 账号区                                 |
|--------------------------------------------------|
| 核心指标卡                                        |
| [今日打开人数] [记录发起率] [完整记录率]          |
| [回填率] [连续记录人数] [提醒转化]                |
|--------------------------------------------------|
| 今日内容状态                                      |
|  日签：已发布 / 待审核 / 失败                     |
|--------------------------------------------------|
| 待处理事项                                        |
|  - 待审核日签 3 条                                |
|  - 提醒任务异常 7 条                              |
|  - 敏感记录处置 1 条                              |
+--------------------------------------------------+
```

### 3.3 交互说明

- 指标卡点击可跳到看板详情
- 待处理事项点击跳到对应模块
- 首页不直接展示原始餐图或原始敏感内容

## 4. 页面 B2：日签生成与审核

### 4.1 页面目标

- 支撑“系统生成为主，人工审核发布”为主的内容供给流程

### 4.2 列表页线框

```text
+--------------------------------------------------+
| 日签生成与审核                                    |
|--------------------------------------------------|
| [生成今日内容] [批量生成未来 7 天]                |
|--------------------------------------------------|
| 筛选：待审核 / 已发布 / 已驳回 / 生成失败          |
|--------------------------------------------------|
| 内容列表                                          |
|  日期 | 预览图 | 文案摘要 | 状态 | 操作           |
|  4/24 | 占位图 | 今天慢一点也没关系 | 待审核 | 查看 |
|--------------------------------------------------|
```

### 4.3 审核详情结构

```text
+--------------------------------------------------+
| 日期 / 状态                                       |
| 插画预览                                          |
| 文案输入区                                        |
| 风险提示区                                        |
|--------------------------------------------------|
| [通过发布] [修改后发布] [驳回重生]                |
+--------------------------------------------------+
```

### 4.4 页面状态

- 生成成功待审核
- 风格风险提示
- 生成失败可重试
- 已发布只读

## 5. 页面 B3：标签与提醒规则

### 5.1 页面目标

- 维护心情标签、饭后感受标签与提醒默认规则

### 5.2 线框结构

```text
+--------------------------------------------------+
| 标签与提醒规则                                    |
|--------------------------------------------------|
| 左侧：规则分组                                    |
|  - 心情标签                                       |
|  - 饭后感受标签                                   |
|  - 提醒默认时长                                   |
|  - 提醒降频规则                                   |
|--------------------------------------------------|
| 右侧：配置区                                      |
|  标签名 / 启用状态 / 排序                         |
|  默认提醒：2 小时                                 |
|  降频：同一用户 24 小时内最多 1 次                |
|--------------------------------------------------|
| [保存规则]                                        |
+--------------------------------------------------+
```

### 5.3 交互说明

- 标签支持启停和排序
- 提醒规则只做最小配置，不展开复杂策略编排
- 修改后只影响新产生的记录和提醒任务

## 6. 页面 B4：记录与转化看板

### 6.1 页面目标

- 让运营和产品看到主闭环是否成立

### 6.2 线框结构

```text
+--------------------------------------------------+
| 记录与转化看板                                    |
|--------------------------------------------------|
| 时间筛选：今日 / 近 7 天 / 近 30 天               |
|--------------------------------------------------|
| 主漏斗                                            |
| 打开首页 -> 发起记录 -> 提交成功 -> 完整记录      |
|--------------------------------------------------|
| 指标区                                            |
| [内容到记录转化] [提醒发送率] [提醒回填率]        |
| [手动回补率] [连续记录 3 天人数]                  |
|--------------------------------------------------|
| 内容表现列表                                      |
|  日签标题 | 曝光 | 点击 | 去记录转化              |
+--------------------------------------------------+
```

### 6.3 交互说明

- 看板只做聚合视图
- 默认不允许下钻查看原始餐图和原始高敏感标签
- 若需看异常案例，需进入受控处置流程

## 7. 页面 B5：敏感记录处置台

### 7.1 页面目标

- 支撑 `隐私边界` 决策中的白名单查看、人工删除和审计要求

### 7.2 线框结构

```text
+--------------------------------------------------+
| 敏感记录处置台                                    |
|--------------------------------------------------|
| 工单编号 / 申请人 / 申请原因                      |
|--------------------------------------------------|
| 记录元数据                                        |
|  record_id / user_id / 时间 / 当前状态            |
|--------------------------------------------------|
| 受控内容查看区                                    |
|  [查看原始餐图]                                   |
|  [查看原始标签]                                   |
|--------------------------------------------------|
| 处置动作                                          |
|  [确认人工删除] [取消提醒任务] [驳回申请]         |
|--------------------------------------------------|
| 审计日志                                          |
|  查看人 / 时间 / 动作 / 原因                      |
+--------------------------------------------------+
```

### 7.3 交互说明

- 该页面不是普通运营入口
- 只有白名单角色可进入
- 每一次查看原始内容和删除动作都要留审计记录

## 8. 后台最小操作流

### 8.1 日签发布流

```text
系统生成日签 -> 进入待审核池 -> 运营审核 -> 发布 -> 前台展示
```

### 8.2 规则维护流

```text
更新标签/提醒规则 -> 保存 -> 仅影响新记录与新提醒任务
```

### 8.3 异常处置流

```text
收到工单/异常 -> 进入敏感记录处置台 -> 查看元数据 -> 必要时受控查看原始内容
-> 删除或关闭 -> 写入审计日志
```

## 9. 本轮原型评审重点

- 后台是否足够小，没有被做成泛运营平台
- 日签审核流是否满足“AI 生成 + 人工审核”路径
- 看板是否只保留验证留存闭环所需的核心指标
- 敏感记录处置是否符合隐私边界决策稿
- 后台是否已经覆盖“前台不开放删除时的人工删除兜底”

## 10. 当前建议

- 先用这套低保真原型完成后台流程评审，再进入接口设计
- 如果 `M0` 最终对权限和删除 SLA 有调整，优先改 `B5 敏感记录处置台`，不要让普通后台页面承接原始敏感内容
